    Lens Behavior (applyTrajLens)
    • No lens selected: Show core datasets (0–4), hide extras (6–8). Cost (5) stays as-is.
    • Core lens (installs/wishlists/players): Show only the mapped datasets, hide all other core + extras. Y-axes yLeft/yRight visible.
    • Extra lens (rating/mentions/sentiment): Show only that dataset, hide everything else. Toggle to yPct or yMentions axis; hide yLeft/yRight.
    • Toggle = show/hide only. No fading or opacity changes. Hidden datasets appear crossed out in legend.
    • Clicking the same lens again deselects it (returns to no-lens default).
    Vs-Yesterday Deltas
    • Computed from trajectory data arrays: data[last] - data[last-1]
    • Cumulative metrics (installs, wishlists): show daily increment as absolute number
    • Daily metrics (players, mentions): show % change
    • Percentage metrics (rating, sentiment): show change in percentage points (pp)
    • Hide when filtered date range does not include today. The delta is meaningless for historical ranges. Either hide the .stat-delta element or replace with a period-appropriate comparison (e.g. last day vs prior day of selected range).

    Cost Array (base data) 
    // allLabels: 109 ISO date strings (Nov 10 – Feb 26)
// allCost:   109 daily cost values in EUR
// Phases:
//   Days 0-13:  Launch burst    (8,000–10,000/day)
//   Days 14-42: Step-down       (7,000 → 3,500/day)
//   Days 43+:   Always-on       (~3,000/day) + periodic promo bursts
    deriveFromCost() Factory 
    function deriveFromCost(ratioPerEuro, opts = {}) {
  const { weekendDip=0.2, noiseFactor=0.08, seedOffset=0 } = opts;
  seed = 200 + seedOffset;
  return allLabels.map((dateStr, i) => {
    const dow = new Date(dateStr).getDay();
    let val = allCost[i] * ratioPerEuro;
    if (dow===0 || dow===6) val *= (1 - weekendDip);  // weekend dip
    val *= (1 + Math.sin((dow-1)/6 * Math.PI) * 0.06); // mid-week peak
    val += (seededRandom()-0.5) * val * noiseFactor;    // noise
    return Math.max(0, Math.round(val));
  });
}
